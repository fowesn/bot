API API documentation

/assignments

Действия конечного пользователя с заданиями

/assignments post get

post

Возвращает случайно выбранное задание для пользователя
Задание случайно выбирается из еще не решенных (также могут применяться другие условия)
Факт выдачи сохраняется

get

Возвращает информацию об одном или нескольких выданных пользователю заданиях

  1. Если не передан ни один необязательный параметр, то метод возвращает статистику пользователя по всем выданным заданиям, подсчитывая число выданных, решенных, нерешенных заданий и заданий, для которых пользователь запросил правильный ответ
    • Выданные = решенные + нерешенные + задания, для которых был запрошен ответ
    • Нерешенным считается задание, на которое НЕ был дан правильный ответ И для которого НЕ был запрошен правильный ответ

    • Решенным считается задание, на которое был дан правильный ответ

  2. Если передан номер задания, то метод возвращает статистику пользователя по заданию (1-3) и информацию о задании(4-6):
    • Количество присланных ответов
    • Правильный ответ прислан (да/нет)
    • Правильный ответ запрошен (да/нет)
    • Тема задания
    • Год публикации задания
    • Номер задания в КИМе
  3. Если передан фильтр, то метод вернет список заданий в соответствии с фильтром
  4. Оба параметра одновременно передать нельзя!

post /assignments

Возвращает случайно выбранное задание для пользователя
Задание случайно выбирается из еще не решенных (также могут применяться другие условия)
Факт выдачи сохраняется

  • Request
  • Response

Body

Media type: application/x-www-form-urlencoded

Type: object

Properties
  • filter: ((integer | string))

    Если параметр не указан, то задание выбирается случайным образом из еще не решенных

    Если параметр имеет тип:

    • string - выбирается задание по указанной теме из еще не решенных
    • integer - выбирается задание под указанным номером в КИМе из еще не решенных

    Example:

    системы_счисления
  • user: required(integer)

    Идентификатор пользователя в используемом сервисе

    Example:

    98765
  • service: required(vk)

    Используемый сервис

    Example:

    vk

HTTP status code 200

Запрос обработан корректно, но не был достигнут ожидаемый результат!

Body

Media type: application/json

Type: any

Examples:

задания кончились:

{
  "status": "a0-2",
  "data": {
    "message": "Stated user has no unassigned problems left!"
  }
}

задания по указанной теме кончились:

{
  "status": "a0-3",
  "data": {
    "message": "Stated user has no unassigned problems with stated problem type left!"
  }
}

задания под указанным номером в КИМе кончились:

{
  "status": "a0-4",
  "data": {
    "message": "Stated user has no unassigned problems with stated exam item number left!"
  }
}
